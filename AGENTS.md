# AGENTS.md
## Commands
- Use `npm run test` for tests.
- Use `npm run build` for type checking and production build validation.
- Use `npm run fetch` to generate `public/articles.json` and `public/sources.json`.
## Project
- This app is a GitHub Pages static app built with TypeScript and Vite.
- Article data is generated by GitHub Actions and stored under `public/`.
- Search runs in the browser against `articles.json`.
- The initial sources are Qiita, Zenn, and Blog.
- Speaker Deck may be added later as another source.
## Security
- Treat all fetched article data as untrusted input.
- Do not inject article titles, tags, URLs, or feed content through `innerHTML`.
- Prefer `textContent`, DOM APIs, and safe attribute assignment for rendered content.
- Validate external URLs before assigning them to links.
- Allow only `https:` URLs for article links unless there is a clear reason otherwise.
- Do not commit secrets, tokens, API keys, or private feed URLs.
- Use GitHub Actions variables for non-secret configuration.
- Use GitHub Actions secrets only when a value must remain private.
- Keep GitHub Actions permissions as narrow as possible.
- Be careful when changing workflows that have `contents: write`.
- Prefer `npm ci` in CI.
- Do not upgrade existing dependencies without explicit approval.
## TypeScript Style
- Keep shared data types in `src/types.ts`.
- Keep pure search logic in `src/search.ts`.
- Keep source normalization and fetch logic in `src/fetchers.ts`.
